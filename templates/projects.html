PurpleLens — Guardrail-First Investigation Pipeline

Open-source POC demonstrating production-grade guardrail architecture

7x reduction in analyst investigation time

The Challenge

Security analysts face repetitive manual event correlation across fragmented telemetry sources, with investigation workflows averaging 30-45 minutes per alert batch. Traditional automation breaks analyst trust when tools produce unreliable outputs.

The Solution

Multi-source telemetry ingestion with normalization, constrained LLM analysis protected by deterministic, semantic, and policy-based guardrails, and full provenance tracking. Each guardrail layer can veto outputs before they reach analysts.

Architecture Approach

Three-layer guardrail enforcement: (1) deterministic validation for known patterns, (2) semantic schema enforcement preventing malformed outputs, (3) policy-based controls enforcing organizational rules.

Demonstrated Impact

  • 7x reduction in investigation time (30-45 min → ~6 min per batch)
  • Zero hallucinated actions through strict guardrail enforcement
  • Full auditability with provenance tracking for compliance
  • Designed for production SOC environments
Note: The GitHub repository contains the proof-of-concept demonstrating the guardrail architecture and evaluation methodology. Operational deployment details remain proprietary.

AI DevSecOps Platform — 5-Layer Security Mesh

Hybrid governance platform detecting threats traditional tools miss

57 adversarial test cases validated

The Problem

Traditional static analysis tools miss AI-specific threats (prompt injection, model poisoning, shadow code modifications), while pure LLM-based security tools lack deterministic guarantees and audit trails required for production CI/CD pipelines.

The Solution

5-layer architecture combining deterministic analysis (AST, compiled regex), semantic taint tracking, operational guardrails (ShellGuard), AI reasoning with Pydantic constraints, and SOC Ledger with cryptographic provenance chain.

Key Innovation: Taint Handshake Protocol

AST provides factual data flows (deterministic), LLM provides semantic intent analysis (contextual). Shadow Code detection flags unauthorized AI modifications through provenance verification. Semantic Drift tracking monitors AI vs. AST disagreements as red-team radar for model blind spots.

Validation Results:
  • Tested across 57 adversarial scenarios
  • Zero false negatives on OWASP LLM Top 10
  • Detects washed secrets & logic bombs
  • CI/CD-ready with sys.exit(1) on CRITICAL
